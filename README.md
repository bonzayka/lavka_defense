# Антиспам-бот для Telegram

Защищает группу от спам-ботов:

1. **3-факторная капча на входе.** Новичок заходит → сразу мут. По шагам:
   - Шаг 1 — математический пример «A + B = ?»;
   - Шаг 2 — случайный вопрос на здравый смысл (буква алфавита, столица,
     цвет снега и т.п. — из пула, чтобы боты не выучили ответ);
   - Шаг 3 — сколько углов у фигуры (треугольник/квадрат/…), ответ цифрой.
   Всё кнопками. Ошибка на любом шаге **или** тишина за 60 сек → **бан навсегда**.
   Действующие **админы** капчу не проходят — пропускаются автоматически.
2. **Распознавание картинок (2 слоя) + зачистка.** Любое присланное фото
   (а также картинка-документ и статичный стикер) проверяется так:
   - **Слой 1 — хеш по базе `photo/`:** точный/пережатый повтор известной
     картинки (похоже **≥ 85%**).
   - **Слой 2 — нейросеть NudeNet (18+):** распознаёт наготу/гениталии на
     **любой новой** картинке, даже которой нет в базе. **Кровь/расчленёнку
     (гор) НЕ распознаёт** — только наготу.

   При срабатывании любого слоя бот:
   - **мутит** нарушителя до решения админа;
   - **удаляет всю связку/альбом** и вообще все сообщения этого юзера
     за последние **5 минут** (что бот успел увидеть — историю чата Bot API
     читать не умеет);
   - постит «Админ, проверь лог нарушения» с кнопками **[Бан]** / **[Размут]**
     (нажать может только админ). Контент смотришь в
     *Управление группой → Недавние действия*.

### Папка `photo/` — база спам-картинок
Кидай в `photo/` файлы со спам-картинками (`.jpg`, `.png`, `.webp`, …) — бот
хеширует их при запуске. Удобнее: просто отвечай на спам-картинку командой
**`/spam`** (см. ниже) — бот сам добавит её в базу без перезапуска. Сравнение
идёт по перцептивному хешу (dhash): ловит ту же картинку, даже пережатую или
изменённого размера. Строгость — `IMAGE_MATCH_PERCENT` в `config.py`
(больше = строже; 100 — точь-в-точь, 85 — пережатая копия, <70 — другая).

## Модерация сообщений

Все сообщения не-админов проходят через фильтры (нарушение → действие из
`config.py`, по умолчанию «удалить»; флуд → мут):

- **Ссылки/инвайты** (`BLOCK_LINKS`), упоминания опционально (`ALLOW_MENTIONS`);
- **Пересылки** (`BLOCK_FORWARDS`), **посты «от имени канала»** (бан канала),
- **`.apk`-файлы**, **премиум-эмодзи** (`BLOCK_PREMIUM_EMOJI`),
- **Антифлуд** (`ANTIFLOOD_COUNT` сообщений за `ANTIFLOOD_SECONDS`),
- **Стоп-слова** + (опционально) **антимат** с фильтром подмены символов
  (гомоглифы: `хyй`, `х.у.й`, `пиzдец`). Антимат **выключен по умолчанию** —
  включается командой `/antimat on` или флагом `ANTIMAT_ENABLED`;
- **Имена вступающих** проверяются на мат/стоп-слова (`CHECK_JOIN_NAMES`).

Режимы: **ночной** (`/night` — ночью пишут только админы), **тихий**
(`/quiet` — уведомления о наказаниях не в группу), удаление сервисных
сообщений вход/выход, приветствие с кнопками (`WELCOME_*`).

Стоп-слова и предупреждения (warn) хранятся в `data.json` и **переживают
перезапуск**. Все настройки — в `config.py`, часть переключается командами.

## Команды (только для админов)

| Команда | Что делает |
|---|---|
| `/spam` | **Ответом** на картинку — добавить в базу спама + наказать автора (без перезапуска). |
| `/reload` | Перечитать папку `photo/`. |
| `/ban` `/unban` | Бан/разбан (ответом или `/ban <id>`). **Со сроком:** `/ban 3 дня`. |
| `/mute` `/unmute` | Мут/размут. **Со сроком:** `/mute 3 часа`. |
| `/warn` `/unwarn` | Предупреждение / снять. На `WARN_LIMIT` (по умолч. 3) — наказание. |
| `/trust` | Доверенный — мимо всех проверок (ответом). |
| `/whitelist` | Разрешить/запретить пользователю ссылки (ответом). |
| `/addword` `/delword` `/words` | Стоп-слова. |
| `/rules` `/setrules` | Показать / задать правила группы. |
| `/report` | **Для всех:** жалоба на сообщение (ответом) → летит админам. |
| `/night` `/quiet` `/antimat` `on\|off` | Режимы. |
| `/settings` `/stats` `/help` `/ping` | Инфо. |

**Тайм-наказания текстом** (ответом на нарушителя, для админов): `мут 3 дня`,
`бан 2 часа`, `размут`, `варн`, `кик`. Шаблон строгий (всё сообщение = слово +
срок), поэтому фразы вроде «я тебе сейчас мут дам» бот **не** триггерят.

Сообщение-команда админа **автоудаляется** после выполнения (`DELETE_ADMIN_COMMANDS`).

### Ещё в этой версии
- **Тайм-наказания** (мут/бан на срок), **доверенные** (`/trust`).
- **Антирейд**: при массовом входе — алерт + локдаун (`RAID_*`), опц. автобан.
- **Ограничение новичков**: первые `RESTRICT_NEWCOMERS_HOURS` ч без ссылок/медиа.
- **Жалобы** `/report` и **правила** `/rules`.
- **Уведомления в ЛС** тем, кто вошёл в панель: о входах (`NOTIFY_JOINS`),
  нарушениях (`NOTIFY_VIOLATIONS`), жалобах (`NOTIFY_REPORTS`) — с id, именем,
  @юзером, текстом и временем до секунды.
- Статистика **переживает перезапуск**.

## Админ-панель в личке бота

Чтобы не писать команды в чат — напиши боту в **личку** `/admin`, введи
**пароль** (`PANEL_PASSWORD` в `config.py`, по умолчанию `Benny` — обязательно
смени!). Откроется панель с кнопками:

- тумблеры всех фич (антимат, ссылки, пересылки, каналы, .apk, премиум-эмодзи,
  антифлуд, антирейд, имена, приветствие, жалобы, ночной/тихий режим, чистки,
  уведомления) — ✅/❌ в один тап;
- **🔢 Числа** — таймаут капчи, пороги антифлуда, лимит варнов, часы для новичков;
- **⚙️ Действия** — что делать за каждый фильтр (delete → warn → mute → ban);
- **📋 Стоп-слова** (добавить/удалить кнопками), **📜 Правила** (изменить),
  **📊 Статистика**, **🔄 Обновить базу**, **❌ Закрыть**.

Доступ держится в памяти и сбрасывается при перезапуске бота (нужно ввести
пароль заново).

## Настройка бота в Telegram

1. Открой [@BotFather](https://t.me/BotFather), создай бота, возьми токен.
2. Вставь токен в `config.py` → `BOT_TOKEN`.
3. Добавь бота в группу и **назначь его администратором** с правами:
   - **Блокировка пользователей** (Ban users)
   - **Ограничение пользователей** (Restrict members)
   - **Удаление сообщений** (Delete messages) — чтобы чистить капчу и «X вошёл в чат».
4. В @BotFather выключи **Group Privacy** (`/setprivacy` → Disable),
   иначе бот не увидит вход новых участников.

## Запуск (Windows)

Зависимости уже установлены в `venv`. Просто:

```bash
cd C:\defense
venv\Scripts\python.exe bot.py
```

Если ставишь с нуля на другой машине:

```bash
py -m venv venv
venv\Scripts\python.exe -m pip install -r requirements.txt
venv\Scripts\python.exe bot.py
```

Проверка живости: напиши боту/в чат `/ping` → ответит `pong`.

## Запуск на Linux-VPS (Ubuntu) — полная версия с ИИ

1. Залей папку с ботом на сервер (например в `/root/defense`) — через `scp`,
   `git clone` или распаковку zip. **Без `venv`** (он соберётся на месте).
2. Из папки бота запусти установку одной командой:
   ```bash
   bash deploy_vps.sh
   ```
   Скрипт ставит системные либы (`libgl1`, `libglib2.0-0` для opencv), создаёт
   `venv` и ставит зависимости (включая NudeNet).
3. Проверь `config.py`: токен; `PROXY` — если VPS **в РФ**, оставь SOCKS5;
   если **за границей**, поставь `PROXY = None` (Telegram доступен напрямую).
4. Разовый запуск для проверки: `./venv/bin/python bot.py` (в чат `/ping`).

### Автозапуск как сервис (systemd)
Чтобы бот стартовал сам и перезапускался при падении:
```bash
# поправь пути в defense-bot.service, если папка не /root/defense
sudo cp defense-bot.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now defense-bot
sudo systemctl status defense-bot      # статус
journalctl -u defense-bot -f           # живой лог
```

> Требования к VPS: Ubuntu 22.04/24.04, **ОЗУ ≥ 1 ГБ** (opencv/onnxruntime),
> диск ≥ 5 ГБ. Системный `python3` подойдёт — под него все колёса есть.

## Настройки (`config.py`)

| Параметр | Что делает |
|---|---|
| `BOT_TOKEN` | Токен от @BotFather |
| `CAPTCHA_TIMEOUT` | Секунд на капчу (по умолч. 60), потом бан |
| `PHOTO_DIR` | Папка с эталонными спам-картинками (по умолч. `photo`) |
| `IMAGE_MATCH_PERCENT` | Порог похожести картинки в % (по умолч. 85; больше = строже) |
| `PURGE_WINDOW_SECONDS` | За сколько секунд чистить сообщения нарушителя (по умолч. 300) |
| `NSFW_ENABLED` | Включить нейросеть-детектор 18+ (по умолч. `True`) |
| `NSFW_MIN_SCORE` | Порог уверенности модели 0..1 (по умолч. 0.5; меньше = строже) |
| `NSFW_BAD_CLASSES` | Какие метки считать нарушением (см. ниже) |
| `DELETE_CAPTCHA_MESSAGE` | Удалять сообщение капчи после ответа/бана |
| `DELETE_JOIN_MESSAGE` | Удалять «X присоединился к группе» |
| `ADMIN_CACHE_TTL` | Сколько секунд кэшировать список админов (по умолч. 300) |
| `LOG_CHAT_ID` | Отдельный чат для уведомлений о нарушениях (по умолч. `None` — постить в группе) |
| `PROXY` | Прокси, если Telegram заблокирован у хоста (по умолч. `None`) |

### Метки NudeNet (для `NSFW_BAD_CLASSES`)
Полный список: `FEMALE_GENITALIA_EXPOSED`, `MALE_GENITALIA_EXPOSED`,
`ANUS_EXPOSED`, `FEMALE_BREAST_EXPOSED`, `BUTTOCKS_EXPOSED`,
`FEMALE_GENITALIA_COVERED`, `FEMALE_BREAST_COVERED`, `BELLY_EXPOSED`,
`FEET_EXPOSED`, `ARMPITS_EXPOSED`, `FACE_*` и т.п. По умолчанию включены только
явные (гениталии/анус/голая грудь). Хочешь жёстче — добавь `BUTTOCKS_EXPOSED`.
Первый запуск инициализирует модель ~1–2 сек; инференс ~50–200 мс на картинку.

## Если не коннектится к api.telegram.org

В РФ доступ к Telegram API часто закрыт. Пропиши прокси в `config.py`:

```python
PROXY = "http://127.0.0.1:8080"          # HTTP-прокси
# или
PROXY = "socks5://user:pass@127.0.0.1:1080"  # SOCKS5
```

Для `socks5://` дополнительно поставь пакет:

```bash
venv\Scripts\python.exe -m pip install aiohttp_socks
```

## Заметки

- Боты, добавленные админом через Bot API (`is_bot=True`), капчу не проходят — их бот пропускает.
- Состояние ожидающих капчу хранится в памяти. Если перезапустить бота, пока
  кто-то «висит» на капче — он останется в муте; перезайдёт → получит капчу снова.
